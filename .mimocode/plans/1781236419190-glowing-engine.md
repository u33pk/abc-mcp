# MCP 服务实现方案

## 目标

基于 ABCDE 反编译能力，实现 stdio 类型的 MCP 服务，为 LLM 提供鸿蒙应用逆向分析工具。

## ABCDE MCP 工具设计

### 核心工具（第一阶段）

| 工具名 | 参数 | 功能 |
|--------|------|------|
| `open_abc` | `path: string` | 打开 ABC/HAP 文件，返回文件信息（版本、类数量等） |
| `list_classes` | `filter?: string` | 列出所有类名，支持正则过滤 |
| `get_class_detail` | `class_name: string` | 获取类详情（父类、字段、方法列表、导出信息） |
| `decompile_class` | `class_name: string` | 反编译指定类的所有方法 |
| `decompile_method` | `class_name: string, method_name: string` | 反编译指定方法 |
| `search_strings` | `pattern: string` | 搜索字符串常量（正则匹配） |
| `disassemble_method` | `class_name: string, method_name: string` | 获取方法的字节码反汇编 |
| `get_method_info` | `class_name: string, method_name: string` | 获取方法详情（参数名、行号、调试信息） |

### 扩展工具（第二阶段）

| 工具名 | 参数 | 功能 |
|--------|------|------|
| `open_hap` | `path: string` | 解析 HAP 包，列出 ABC 文件和资源 |
| `search_resources` | `pattern: string, type?: string` | 搜索资源索引 |
| `get_xrefs_to_method` | `class_name: string, method_name: string` | 查找方法的调用者 |
| `list_strings_in_method` | `class_name: string, method_name: string` | 列出方法中引用的字符串 |

### 关键实现细节

#### 1. MCP 协议（stdio）

```kotlin
// 读取 stdin，每行一个 JSON-RPC 请求
// 写入 stdout，每行一个 JSON-RPC 响应
fun run() {
    val reader = System.`in`.bufferedReader()
    val writer = System.out.bufferedWriter()
    
    while (true) {
        val line = reader.readLine() ?: break
        val request = Json.parseToJsonElement(line).jsonObject
        val response = handleRequest(request)
        writer.write(Json.encodeToString(response))
        writer.newLine()
        writer.flush()
    }
}
```

#### 2. 会话管理

```kotlin
class SessionManager {
    private val sessions = mutableMapOf<String, AbcBuf>()
    
    fun open(path: String): AbcBuf {
        val file = File(path)
        val buf = FileChannel.open(file.toPath())
            .map(FileChannel.MapMode.READ_ONLY, 0, file.length())
        val abc = AbcBuf(path, wrapAsLEByteBuf(buf))
        sessions[path] = abc
        return abc
    }
    
    fun get(path: String): AbcBuf? = sessions[path]
}
```
