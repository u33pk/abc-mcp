# 函数调用合并实现方案

## 问题描述

当前反编译输出：
```javascript
_acc_ = AtkTsGlobal.print;
_acc_ = this._acc_(_acc_);
```

期望输出：
```javascript
_acc_ = AtkTsGlobal.print(_acc_);
```

## 根因分析

| 问题 | 位置 | 说明 |
|------|------|------|
| `when` 分支死代码 | `StructuredToJs.kt:142-158` | 第一个分支匹配所有 `AssignReg(ACC, ...)`，合并分支永远不可达 |
| 优化器无跨指令合并 | `OptimizationPasses.kt` | `ExpressionPropagationPass` 只做单指令内联，不合并两条指令 |
| `CallAcc` 隐式读 ACC | `IrOp.kt:340` | `CallAcc.read()` 返回 ACC，但优化器只解析 `overrideThis` 和 `args` |
| 代码生成硬编码 | `StructuredToJs.kt:237` | `generateExpression(CallAcc)` 固定输出 `this._acc_(...)` |

## 实现方案

### 方案：在优化器层合并（推荐）

在 `ExpressionPropagationPass` 中检测模式并重写 IR：

**检测模式**：
```
AssignReg(ACC, X)           // X = ObjField.Name / LoadReg / 等
AssignReg(ACC, CallAcc(args, overrideThis))
```

**重写为**：
```
AssignReg(ACC, CallWithTarget(X, args, overrideThis))
```

### 实现步骤

#### Step 1: 新增 `CallWithTarget` IR 节点

**文件**: `modules/abcde/src/commonMain/kotlin/me/yricky/oh/abcd/decompiler/behaviour/IrOp.kt`

在 `Expression` 子类中新增：
```kotlin
class CallWithTarget(
    val target: Expression,      // 被调用的目标（如 ObjField.Name(GLOBAL, "print")）
    val args: List<FunSimCtx.RegId>,
    val overrideThis: FunSimCtx.RegId? = null
) : Expression {
    override fun read(): Sequence<FunSimCtx.RegId> = 
        args.asSequence() + (overrideThis?.let { sequenceOf(it) } ?: emptySequence())
    override fun effected(): Sequence<FunSimCtx.RegId> = sequenceOf(FunSimCtx.RegId.ACC)
}
```

#### Step 2: 在 `ExpressionPropagationPass` 中添加合并逻辑

**文件**: `modules/abcde/src/commonMain/kotlin/me/yricky/oh/abcd/decompiler/structure/OptimizationPasses.kt`

在 `transform` 方法中，当处理 `AssignReg(ACC, CallAcc(...))` 时：
1. 检查前一条指令是否为 `AssignReg(ACC, X)`
2. 如果是，将当前指令重写为 `AssignReg(ACC, CallWithTarget(X, args, overrideThis))`
3. 将前一条指令标记为删除（或替换为 NOP）

需要在 pass 中维护 `prevOp` 状态来实现跨指令模式匹配。

#### Step 3: 更新代码生成

**文件**: `modules/abcde/src/commonMain/kotlin/me/yricky/oh/abcd/decompiler/structure/StructuredToJs.kt`

1. `generateExpression` 添加 `CallWithTarget` 分支：
```kotlin
is IrOp.CallWithTarget -> {
    val targetStr = generateExpression(exp.target)
    val argsStr = exp.args.joinToString { generateRegId(it) }
    "$targetStr($argsStr)"
}
```

2. 删除 `generateLinearBlock` 中的死代码 `accSource` 追踪逻辑（已无用）

#### Step 4: 更新 `replaceReg` 支持

**文件**: `modules/abcde/src/commonMain/kotlin/me/yricky/oh/abcd/decompiler/behaviour/suger.kt`

为 `CallWithTarget` 添加 `replaceReg` 支持。

#### Step 5: 更新优化器其他 Pass

**文件**: `OptimizationPasses.kt`

- `AlgebraicSimplificationPass`: 无需改动（不涉及 CallAcc）
- `CopyPropagationPass`: 需要处理 `CallWithTarget` 的 `read()`/`effected()`
- `DeadCodeEliminationPass`: 需要识别合并后的指令
- `AccCopyPropagationPass`: 需要处理 `CallWithTarget`
